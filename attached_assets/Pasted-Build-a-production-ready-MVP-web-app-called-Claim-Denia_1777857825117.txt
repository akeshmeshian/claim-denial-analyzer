Build a production-ready MVP web app called “Claim Denial Analyzer.”

Purpose:
Users upload an insurance denial letter and insurance policy PDF. The app extracts the text, analyzes the denial using OpenAI, and generates a structured educational report with possible coverage arguments, policy citations, weaknesses, and a draft message to the insurer.

Use this stack:
- Next.js with TypeScript
- Tailwind CSS
- Supabase for database, auth, and file storage
- Stripe Checkout for payments
- OpenAI API for analysis
- PDF text extraction library for uploaded PDFs
- Deployable to Vercel

Important legal positioning:
This is NOT legal advice, NOT insurance advice, and NOT a substitute for a licensed attorney, public adjuster, or insurance professional. Display this disclaimer on the landing page, upload page, checkout page, and report page.

OpenAI behavior:
Create an API route /api/analyze that:
1. Receives report ID
2. Extracts text from the denial letter PDF
3. Extracts text from the policy PDF
4. Sends the extracted text to OpenAI
5. Saves preview and full report to Supabase
6. Returns preview to user

AI prompt:
Use this exact prompt structure:

“You are an insurance document analysis assistant.

You are analyzing an insurance denial letter and insurance policy for educational purposes only.

Rules:
- Only use the denial letter and policy text provided.
- Do not invent policy language.
- Do not claim something is covered unless the provided policy language supports the possibility.
- If policy wording is missing, unclear, or incomplete, say so.
- Do not give legal advice.
- Do not tell the user they definitely have coverage.
- Use cautious language such as ‘possible argument,’ ‘may support,’ ‘could be relevant,’ and ‘needs review.’
- Cite exact wording from the uploaded policy whenever possible.
- Identify both strengths and weaknesses.
- Write in plain English.

User claim type:
[claim_type]

User claim description:
[claim_description]

Denial letter text:
[denial_text]

Policy text:
[policy_text]

Return JSON with this structure:
{
  "denial_summary": "",
  "key_policy_language": [
    {
      "quote": "",
      "why_it_matters": ""
    }
  ],
  "possible_coverage_arguments": [
    {
      "title": "",
      "argument": "",
      "supporting_policy_language": "",
      "strength_level": "Weak / Moderate / Strong"
    }
  ],
  "insurer_exclusions_or_limitations": [
    {
      "issue": "",
      "why_it_may_hurt_the_claim": ""
    }
  ],
  "weaknesses_or_missing_information": [
    ""
  ],
  "questions_to_ask_adjuster": [
    ""
  ],
  "draft_message_to_insurer": "",
  "educational_disclaimer": "This report is educational document analysis only. It is not legal advice, insurance advice, or a substitute for a licensed attorney, public adjuster, or insurance professional."
}”

Important:
If PDF text is too long for one OpenAI request, chunk the policy text and first identify relevant sections, then use only the relevant sections for the final analysis.
